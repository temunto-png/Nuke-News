# 設計書: せっかくだから俺はこのニュースで抜くぜ

**作成日:** 2026-04-07
**ステータス:** v2.0 ブラッシュアップ済み
**リポジトリ:** https://github.com/temunto-png/Nuke-News.git

---

## 1. サービス概要

ニュースをAIが分析し、意外な視点でAVジャンル/作品に紐付ける笑えるWebアプリ。
毎日5件を自動キュレーション。**Xの自動投稿を軸にSNSで拡散**し、FANZAアフィリエイトで収益化する。

**コアバリュー:**
- ニュースとAVの予想外な組み合わせによる笑い
- 「答えを見るまで来ないとわからない」設計でSNS流入を最大化
- 運用コストほぼゼロの完全自動化

---

## 2. ターゲット

- 日本語ユーザー、18歳以上
- スマートフォンメインで閲覧
- Xでネタコンテンツをシェア・消費するユーザー層

## 3. 収益設計（改訂）

### 収益レバー

| レバー | 仕組み | 単価感 |
|--------|--------|--------|
| **FANZA 単品アフィリエイト** | 作品リンク経由の購入/レンタル | 低〜中 |
| **FANZA 月額サービス誘導** | 各カード下部に「月額で見放題」CTA追加 | 高（月額2,980円〜の成果報酬） |

### 計測設計（必須）

- アフィリエイトURLに**日付・ニュースID・CTAタイプ**をUTMパラメーターで付与
  ```
  https://al.dmm.com/?... &utm_source=nukenews&utm_medium=card&utm_campaign=2026-04-07-item1-monthly
  ```
- Vercel Analytics（無料枠）でPV・クリック率を日次計測
- 週次で「どのジャンル組み合わせがCVRが高いか」をJSONログに蓄積 → AIの選定プロンプトにフィードバック（学習ループ）

## 4. 全体アーキテクチャ（改訂）

```
[GitHub Actions Cron: 毎日 AM7:00 JST]
        │
        ▼
[1] RSS取得
    NHK / Yahoo!ニュース / livedoorニュース（複数ソースでフォールバック）
    → 当日の主要ニュース一覧（最大50件）
        │
        ▼
[2] AI一括処理（Claude Haiku, 1プロンプト）
    全記事タイトル+概要（RSS description）を送信
    → 以下を一度に返す：
      - 「AVジャンルと面白い接続が作れる5件」の選定
      - 各件のジャンルキーワード（意外性 or 感情トーンの面白い方）
      - 「なぜこの作品を選んだか」の理由文（1〜2文）
      - Xシェア用ティザーテキスト（作品名は含めない）
        │
        ▼
[3] FANZA Affiliate API 検索（5件並列）
    ジャンル/キーワードで作品検索
    → 成果報酬率の高い作品を優先（新作・独占配信フラグ）
    → 上位1件のサムネURL・作品タイトル・アフィリエイトURL取得
        │
        ▼
[4] JSONファイル生成
    /data/YYYY-MM-DD.json に書き出し
        │
        ├── [5a] Vercel Deploy Hook を直接叩いてデプロイトリガー
        │         （git pushなし → 無限ループ回避）
        │
        └── [5b] X API で自動投稿（5件を順次投稿、30分間隔）
                  ティザーテキスト + サイトURL
```

### バッチ失敗時のフォールバック

- バッチ失敗 → 前日のJSONデータを `/data/latest.json` として表示し続ける
- GitHub Actions 失敗時は Slack/メール通知（GitHub Actionsの標準機能）


### SNSシェア設計（ティザー型）

作品名・ジャンルは**シェアテキストに含めない**。サイトに来るまでわからない状態を作ることでCTR最大化。

```
【シェアテキスト例（AIが自動生成）】
「日銀が金利を据え置いた」— このニュースで選ばれた作品、絶対わからんと思う。

せっかくだから俺はこのニュースで抜くぜ
https://nukenews.jp/2026-04-07#1
```

### OGP画像（`@vercel/og` で動的生成）

- ニュースタイトルを大きく中央配置
- 作品サムネを背景に薄く（モザイクなし、Xが自動でセンシティブ扱い → 逆にタップ誘発）
- サイト名をフッターに

### SEO

- 各日付ページに固有URL（`/2026-04-07`）
- `<title>` = `「[ニュースタイトル]」で抜く — せっかくだから俺はこのニュースで抜くぜ`
- `<meta description>` に理由文の冒頭を使用
- 過去アーカイブが長期的な検索流入資産になる

## 6. X自動投稿設計（新規追加）

### 目的

フォロワーが増えるほど流入が自動増加する構造。コンテンツ自体がネタなので拡散親和性が高い。

### 投稿フォーマット

```
【本日の5本 📰→🔞】

①「日銀、金利据え置き」
②「○○選手が電撃移籍」
③「△△大臣が辞任表明」
④「◇◇の物価が過去最高に」
⑤「□□の新技術が世界初」

それぞれどのジャンルになったか、想像できる？👇
https://nukenews.jp/2026-04-07

#ヌケニュース
```

### 実装

- GitHub Actions から X API v2（OAuth 2.0）でPOST
- 投稿は1日1回（全5件まとめて）
- Xの無料APIティアで月1,500ツイートまで投稿可能（十分）

## 9. バッチ処理詳細（改訂）

```yaml
# .github/workflows/daily.yml
# schedule: '0 22 * * *'  # UTC 22:00 = JST 07:00
```

### 処理ステップ

1. **RSS取得:** NHK・Yahoo!・livedoorから当日記事を取得（最大50件）。1つ失敗しても次のソースで継続
2. **AI一括処理（Haiku, 1プロンプト）:** 選定 + ジャンル決定 + 理由文 + シェアテキストを同時生成。JSONで返させる
3. **FANZA API検索（5件並列）:** ジャンルキーワードで検索。新作・独占配信フラグがある作品を優先スコアリング
4. **サムネ取得確認:** 404の場合はFANZAのデフォルト画像URLに差し替え
5. **JSON生成:** `/data/YYYY-MM-DD.json` と `/data/latest.json` を出力
6. **Vercel Deploy Hook:** URLにPOSTしてデプロイトリガー（git pushなし）
7. **X自動投稿:** 5件まとめ投稿
8. **失敗通知:** いずれかのステップ失敗でGitHub Actions標準のメール通知
